## Rate Limit 처리 전략
```
요청 전 → 슬라이딩 윈도우 체크 (60초 내 38회 이하 유지)
                │
         한도 근접 시
                │
                ▼
         자동 sleep (필요한 초만큼)
                │
         요청 실행
                │
         HTTP 429 수신 시
                │
                ▼
         Retry-After 헤더 파싱
                │
                ▼
         대기 후 tenacity 재시도 (최대 3회)
```

**API 호출 수 예측 (코인 200개 기준)**

| 엔드포인트 | 호출 전략 | 예상 횟수 |
|-----------|----------|----------|
| open-interest | 20개씩 묶음 | 10회 |
| funding-rate | 20개씩 묶음 | 10회 |
| long-short-ratio | 20개씩 묶음 | 10회 |
| liquidation-history | 20개씩 묶음 | 10회 |
| **합계** | | **40회** |

> ⚠️ 코인 200개를 분당 40회 이내로 처리하려면
> 각 청크 사이에 `sleep(1.5초)` 삽입 필수
